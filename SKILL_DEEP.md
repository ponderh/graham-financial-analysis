# Deep Annual Report Analysis — 深度年报解析框架

## 定位

本框架是格雷厄姆分析Skill的**深度增强模块**，专门解决"年报PDF海量信息如何充分利用"的问题。

数据源覆盖四类文档：
| 文档 | 频率 | 解析深度 |
|------|------|---------|
| **年报PDF** | 每年1次 | 全量深度解析（8章节） |
| **半年报PDF** | 每年2次 | 核心章节解析 |
| **季报PDF** | 每季度1次 | 关键指标追踪 |
| **公告PDF** | 不定期 | 事件驱动补充 |

**架构原则：年报为主干，半年报为补充，季报为追踪，公告为触发。**

## 四专家 + Critic 框架

### 角色分工

| 角色 | 专家 | 核心职责 | 关注问题 |
|------|------|---------|---------|
| 财务审计师 | **Graham** | 财务附注深度 + 会计政策审查 | 利润质量、会计激进/保守 |
| 商业本质分析师 | **Buffett** | 护城河 + 管理层诚信 + 内在价值 | 商业模式、管理层是否可信 |
| 风险管理师 | **Marks** | 风险识别 + 第二层思维 + 周期定位 | 风险定价、不对称性 |
| 量化评估员 | **Critic** | 质疑三位专家 + 补充被忽视数据 | 逻辑漏洞、股东掏空迹象 |
| **协调器** | **Orchestrator** | 汇总四专家结论，生成最终报告 | — |

### 每类文档的解析内容

#### 年报（完整8章节）

**第一章：财务附注（Graham负责）**
- 收入确认政策（是否激进？）
- 应收账款坏账计提（预期信用损失模型细节）
- 存货计价方法（加权平均/先进先出？）
- 固定资产折旧（年限是否合理？）
- 无形资产摊销
- 研发费用资本化条件（资本化时点？）
- 金融工具减值政策
- 关联方交易完整明细

**第二章：业务本质（Buffett负责）**
- 董事会报告（MD&A）定性分析
- 主要产品/收入构成
- 核心竞争力描述
- 未来发展战略
- 员工结构（技术人员占比）
- 控股参股公司情况

**第三章：风险因素（Marks负责）**
- 重大风险提示章节
- 或有事项（担保/诉讼/仲裁）
- 资产负债表日后事项
- 借款情况
- 应收账款质量（账龄+逾期）
- 存货跌价准备
- 政策风险（集采/医保谈判）
- 员工薪酬和人才风险

**第四章：治理与股东（Critic负责）**
- 前十大股东变化（大资金动向）
- 前五大客户/供应商集中度
- 募资使用情况（IPO定增资金投向）
- 商誉及减值测试
- 高管薪酬和激励机制
- 现金分红情况
- 少数股东权益异常
- 股份支付细节

**第五章：财务报表重构（Graham负责）**
- 利润表还原（营归Py=扣非后真实利润）
- 现金流质量（经营CF vs 净利润）
- 资产负债表健康度
- ROE分解（杜邦分析）
- 应收款项质量深度分析

**第六章：竞争格局（Buffett负责）**
- 同行对比（规模/毛利率/研发投入）
- 市场份额数据
- 竞争壁垒描述
- 技术迭代风险
- 替代品威胁

**第七章：周期定位（Marks负责）**
- 行业生命周期位置
- 政策周期（集采/医改）
- 经营杠杆分析
- 估值在周期中的位置

**第八章：综合质疑（Critic负责）**
- 综合三位专家观点的逻辑一致性
- 识别最被忽视的风险
- 识别最被忽视的机会
- 最终操作建议

#### 半年报（简化版，4章节）
财务附注 + 业务本质 + 风险因素 + 关键财务数据

#### 季报（精简版，2章节）
关键指标（营收/净利/CF）+ 经营趋势 + 重大变化

#### 公告（事件驱动）
| 公告类型 | 触发解析内容 |
|---------|------------|
| 业绩预告/快报 | 实际 vs 预告差异，业绩修正原因 |
| 股权激励 | 行权条件/价格/摊销/激励效果 |
| 并购重组 | 标的质地/业绩承诺/商誉风险 |
| 监管问询 | 问询内容→公司答复→隐藏问题 |
| 股权变动 | 大股东增减持/机构进出 |
| 担保/诉讼 | 或有负债量化风险 |
| 分红方案 | 分红率/持续性/能力vs意愿 |

## 使用方式

```bash
# 完整深度分析（年报+半年报+季报+公告）
python3 parse_deep.py 688351 ./output --type full

# 年报专项（最快）
python3 parse_deep.py 688351 ./output --type annual

# 公告触发（业绩预告后）
python3 parse_deep.py 688351 ./output --type announcement --ann-type earnings预告

# 四专家并行分析
python3 parse_deep.py 688351 ./output --experts all

# 单一专家分析
python3 parse_deep.py 688351 ./output --experts Graham,Buffett
```

## 关键设计决策

1. **为什么用四专家而不是一个Agent？**
   - 单一Agent容易陷入"确认偏误"——先下结论再找证据
   - 四专家天然立场对立（质量派vs成长派vs风险派vs质疑派）
   - 辩论结构让错误判断更容易暴露

2. **为什么四类文档都要？**
   - 年报太滞后（每年1次，4个月后发布）
   - 半年报验证趋势
   - 季报追踪最新动态
   - 公告揭示事件驱动变化

3. **为什么输出标准化JSON而不是纯文本？**
   - 便于程序化评分和排序
   - 便于历史对比（跨年/跨公司）
   - 便于量化打分（维度平均→综合评分）
